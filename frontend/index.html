Frequently Asked Questions

Quick answers about shared ledgers, sign-in, event access, and summaries.

What problem does NOTEPAY solve?
It replaces a physical donation register. Multiple collectors can update one shared ledger at the same time, while organizers see live totals without carrying a notebook everywhere.
How does login and session security work?
Every new login requires mobile OTP verification. Sessions can stay active on each device, multiple devices are supported, and signing out requires OTP verification on the next login.
What happens when an event is deactivated?
Collectors are locked out immediately and the current event code becomes permanently invalid. The organizer can still view data in read-only mode and must generate a new code after reactivation.
What is shown in the Summary tab?
Summary shows totals only: total donations collected, total expenses spent, balance remaining, number of donors, and number of expense entries. Individual rows stay in their own tabs.
What are Visited Events?
Public events opened through a shared view link can appear under Visited Events, so users can return to recently viewed public ledgers from the dashboard.
Can members send receipts to donors?
Yes. Donation entries include a receipt-sharing action that creates a clean PDF receipt with donor name, amount, date, collector, and event details.
Can organizers download event records?
Yes. Organizers can download a PDF report for an event with donation rows, expense rows, custom fields, and financial summary totals.
Can organizers control what public viewers see?
Yes. Public access can be turned on or off, and donation or expense table visibility can be controlled so at least one table remains visible.
